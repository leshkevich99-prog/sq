## 🛠️ 02. Матрица Бизнес-Процессов (Interaction Matrix)

| Действие пользователя | Изменение в БД | Триггер уведомления | Изменение в UI |
| :--- | :--- | :--- | :--- |
| **Создание заказа** | Новый док в `requests` | Пуш админу в Telegram | Появление в колонке «Новые» |
| **Назначение пилота** | `pilotId` + статус `accepted` | Пуш пилоту в Telegram | Задача в активном списке пилота |
| **Нажатие «Выехал»** | Статус `driving` | Обновление в чате | Статус-бар клиента «В пути» |
| **Нажатие «Завершил»**| Статус `review` + медиа | Пуш админу «На проверке» | Перенос в колонку «Проверка» |
| **Закрытие задачи** | Статус `completed` + квота | Пуш клиенту «Готово» | Перенос в «Архив» |

## 🛰️ 03. Техническая Карта API и Данных

### Основные Эндпоинты (API Reference)
- `POST /api/auth/me` — Валидация сессии и получение профиля.
- `PUT /api/requests/:id` — Обновление статуса и медиа-данных задачи.
- `POST /api/upload-proxy` — Безопасная прокси-загрузка фото в облако.
- `POST /api/notifications/send` — Рассылка Telegram-пушей по ролям.

### Структура БД (Conceptual Schema)
- `users`: Профили, роли (admin/pilot/client), лимиты квот.
- `requests`: Вся логика задач (статусы, адреса, координаты, медиа-ссылки).
- `cars`: Реестр автопарка с привязкой к владельцам.
- `billing`: Журнал расходов и транзакций по каждой задаче.

## 🛡️ 04. Техническая устойчивость (Architecture)
- **Visibility Resume**: Обновление токенов при возврате из фона. Сессия не «протухает».
- **Image Parallel Batching**: Загрузка до 10 фото одновременно. Скорость загрузки отчета — колоссальная.
- **Safe Area Support**: Дизайн адаптирован под iPhone (Dynamic Island, нижние бары).
- **Offline Logic**: Если связь пропала, приложение кэширует медиа-очередь до появления сети.
