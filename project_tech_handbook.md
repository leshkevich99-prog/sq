## 02. Протоколы Взаимодействия (API Flow)

### 📤 2.1. Загрузка медиа (Upload Flow)
- **Запрос**: `POST /api/upload-proxy`
- **Тело (JSON)**:
  ```json
  {
    "base64Data": "base64_string...",
    "fileName": "requests/ID/before_timestamp.jpg",
    "token": "FIREBASE_ID_TOKEN",
    "bucket": "squadra-app.appspot.com"
  }
  ```
- **Логика**: Сервер проверяет токен, переправляет файл в Firebase Storage и возвращает `downloadURL`. Если загрузка падает — возвращается код ошибки 500 с описанием причины.

### 🔄 2.2. Смена статуса (Status Update Flow)
- **Запрос**: `PUT /api/requests/:id`
- **Payload**: `{ "status": "review", "photoMetadata": { ... } }`
- **Сторона сервера**: После обновления документа в БД бэкенд автоматически дергает функцию `notifyAdmins`, которая рассылает Telegram-пуши.

## 04. Справочник Статус-кодов и Ошибок

- **401 Unauthorized**: Истек токен Firebase или JWT-кука. Реакция: авто-вызов `refreshAuth`.
- **403 Forbidden**: Попытка пилота редактировать задачу другого пилота.
- **404 Not Found**: Задача была удалена или ID неверный.
- **500 Server Error**: Ошибка записи в Firestore или сбой Telegram Bot API.

---

## 05. Оптимизация Производительности (Latency)

1. **Firestore Snapshots**: Мы используем подписки на обновления документов. Это исключает лишние сетевые запросы при каждом действии — данные «прилетают» сами.
2. **Image Batching**: Параллельная загрузка фото через `Promise.all` сокращает время простоя процессора и канала связи.
3. **Memoization**: Тяжелые компоненты (карточки задач в Kanban) кешируют свои состояния, чтобы скроллинг оставался плавным даже при 50+ активных задачах.

## 🛡 06. Секреты стабильности мобильной версии

- **Safe Area Insets**: Дизайн учитывает отступы на iPhone (челки и нижний бар-индикатор).
- **Pointer Events Optimization**: Убраны все задержки кликов, характерные для браузера. Кнопки реагируют мгновенно (Touch-First).
- **Background Persistence**: Слушатель `visibilitychange` — это страховка, которая обновляет связь за миллисекунды до того, как вы начнете ввод данных.
